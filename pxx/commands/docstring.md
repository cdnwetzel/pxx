# /docstring — Add a concise docstring

Add or refine docstrings on public functions and classes. Use the style from the codebase (one-line summary, optional longer description if the WHY is non-obvious). Do not add unnecessary docstrings to internal helpers or simple functions where the signature is self-explanatory.

## Guidelines
- Public functions and classes only
- One line for simple cases
- Multi-line for complex behavior (only when WHY is non-obvious)
- Follow the codebase style exactly
